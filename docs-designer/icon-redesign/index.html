Elysium Designer: Icon Redesign Catalog

Every toolbox / toolbar / viewport glyph, with its identity and purpose. Redesign each in your editor, then hand the new art back to be imported. 42 icons in 4 families.

How to redesign & hand back

  • One file per icon, named <kind>.svg (keep the exact id, e.g. tool_select.svg): that id maps the art to the right slot.
  • Start from TEMPLATE.svg (48×48 artboard, center + safe-area guides) or edit the matching current/<kind>.svg as a reference of what the icon means.
  • Deliver the folder of .svg files (or paste the path d strings). I wire each into the @icon_painter("<kind>") registry: or, on request, I can add a generic SVG loader so the folder drops in directly.

Hard requirements (so the art imports & renders correctly)

  • Path data only. The renderer parses the SVG path d string via Skia parse_path::from_svg: not a full SVG document. OK: <path d="…"> using M L H V C S Q T A Z (absolute or relative). Flatten to paths: <rect> <circle> <line> <polygon> <polyline> (a stand-alone dot may use <circle>).
  • No unsupported SVG features. No transform= (bake it into the coordinates), no <g>, gradients, filters, masks, clip-paths, patterns, markers, or stroke-dasharray. Convert any <text> to outlines.
  • 48×48 artboard, centred. Use a viewBox="0 0 48 48"; center the glyph on (24, 24); keep all art inside the 4 … 44 safe area (≈40×40). The app scales the artboard into toolbox/toolbar buttons.
  • Single color, recolourable. Draw in one color: use currentColor (or a single flat hex). Don't bake brand/palette colours; the theme tints every icon to the foreground and applies the iris accent only to the active item. If you truly need depth, a second tone at ~60% opacity of the same color is the max.
  • Monoline weight. Target stroke-width ≈ 1.6 in artboard units; prefer strokes over fills. Caps & joins render round automatically (you don't need to set them, but stroke-linecap/linejoin="round" is fine).
  • Keep it legible at 18–22px. Icons render small; avoid <1px detail, dense hatching, or text. Aim for a clear silhouette that reads at toolbar size.
Reference assets live beside this file: current/*.svg (today's glyphs), TEMPLATE.svg, manifest.json (kind → category + purpose).
