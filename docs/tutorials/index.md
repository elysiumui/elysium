# Tutorials

Build-something walkthroughs beyond the four lead demos in
[Getting Started](../getting-started/index.md). Each tutorial
produces a finished, shippable app.

## The tutorials

| Tutorial | Time | Difficulty | What you build |
|---|---|---|---|
| [Borderless music widget](borderless-music-widget.md) | 25 min | Beginner | Star-shaped widget variant of Stylized Music Player, no visualizer |
| [Particle pet](particle-pet.md) | 35 min | Intermediate | Floating cursor-following companion with Spring physics |
| [Command palette launcher](command-palette-launcher.md) | 25 min | Intermediate | Global hotkey rounded borderless launcher with fuzzy search |
| [AI-assisted redesign](ai-assisted-redesign.md) | 20 min | Intermediate | Aether redesigns Aurora Clock with "make it sunset themed" |
| [Aurora Clock Pro](aurora-clock-pro.md) | 60 min | Advanced | Multi-window timer + stopwatch + settings, evolved from the lead demo |
| [Weather widget](weather-widget.md) | 30 min | Intermediate | Borderless widget using `webview` for API fetch |

## How to read these

Each tutorial:

- Pitches the result with a hero image at the top.
- Lists prerequisites (which lead demo to do first).
- Walks through code in copy-pasteable chunks.
- Ends with packaging via `elysium pack`.

For one-question "how do I X" answers, the [Recipes](../recipes/index.md)
cookbook is the better entry.

## Submit a tutorial

Tutorials live in `docs/tutorials/`. Add one by opening a PR with
a finished example in `examples/` plus the markdown walkthrough.
